# 内容选题助手

把"选题"从拍脑袋变成一套可复用的发散 + 评估流程。适用于公众号、视频号、小红书、知乎、播客等内容账号的选题策划。

## 一、核心定位（先讲清一件事）

> 全媒体时代，**选题策划比写更重要**。好的选题 = 在诸多素材中挖到好题材与好视角，经加工后传播更广、也更有意义。

选题不是"写什么"，而是"为什么是它、从什么角度、给谁看"。

## 二、选题基础三问（每次策划前先锚定）

在发散选题前，必须先回答这三个问题；用户没给就主动问，不要跳过：

1. **账号定位**：这个账号是什么类型？（如运营类、职场类、母婴类、数码类……）
2. **人格化**：内容要呈现什么风格 / 价值观 / 调性？（解决"我是谁"的问题）
3. **为用户服务**：帮**哪些用户**在**什么场景**解决**什么问题**，或让他们知道**什么新东西**？
   - 模板句式：「帮『＿＿用户』在『＿＿场景』解决『＿＿问题』/ 知道『＿＿新东西』」

示例（运营类账号）：用户是运营人员，在日常工作中把"运营小白"变成"高阶运营人"——这就是选题方向。

## 三、横纵发散法（生成选题池的主干）

沿两个方向把领域拆开，选题自然涌现：

- **横向 = 逻辑关联板块**：列出该领域涉及的几大板块。
  - 例：运营 → 运营基础能力 → 新媒体运营 → 活动运营 → 内容运营 → 用户运营
- **纵向 = 垂直细分阶段**：把每个板块再按生命周期拆阶段。
  - 例：活动运营 → 活动准备 → 活动策划 → 活动预热 → 活动执行 → 活动复盘
- **用法**：
  - **单点成选题**：「活动运营的目标如何确立」
  - **组合成选题**：把多个点结合，做对比 / 流程 / 避坑类内容

## 四、10 大选题绝招（发散角度库）

当用户说"给我想几个选题"时，逐条套用，每条给「触发关键词 + 可直接套用的标题模板」。详见 `references/选题评估与工具箱.md` 的速查卡，这里给判断逻辑：

1. **追热点** — 触发：朋友圈/同事都在聊的事。动作：勤看各类排行榜，找可做的切入点。
2. **反热点** — 触发：大家都在做一件事时，关注"谁没做？为什么不做？"找第二落点。
3. **找变化** — 触发：日常/季节/天气/趋势的新变化。随时想"这个变化能做什么选题"。
4. **想经历** — 触发：难忘的个人经历。用第一人称输出，引发共鸣、写得生动不生硬。
5. **有争议** — 触发：模糊地带、有人气的地方。适当抓争议引发讨论。
6. **总结套路** — 触发：透过热点抓套路，把复杂事件转成简单可复用的方法论。
7. **发现未知** — 触发：用户的好奇心/窥伺欲（"未知 + 好奇 + 窥伺 = 爆款代名词"）。
8. **用户痛点** — 触发：核心用户的需求痛点，内容往那使劲（不追求让所有人满意）。
9. **UGC（用户生产内容）** — 触发：向粉丝征集、整理投稿，全民参与创作（粉丝少时仍需编辑介入）。
10. **换个形式** — 触发：同一个老选题，换一种内容形式 / 载体 / 呈现方式重新表达，即新选题。（原 PPT 此页为图示，含义据标题推断）

## 五、选题价值评估：5 标准雷达打分法

对候选选题打分（每维 1–5 分），画雷达图，越均衡完整价值越高：

| 维度 | 含义 | 高分信号 |
|------|------|----------|
| 热度 | 当下关注量 | 大家在聊、搜索量涨 |
| 传播度 | 易被转发 | 情绪强、易共鸣 |
| 话题性 | 能引发讨论 | 有观点冲突空间 |
| 相关度 | 与账号定位匹配 | 命中核心用户 |
| 风险度 | **反向计**，风险越低分越高 | 无合规/舆论隐患 |

> 追热点的"切入点角度"也用同类 5 标准判断，只是把"热度"换成"覆盖度"（能覆盖多少相关人群）。

**判定规则**：5 维都及格（≥3）且无明显短板（风险度≥4）的选题优先做；某维极低（如相关度=1）则放弃或改角度。

## 六、标准工作流（agent 执行步骤）

收到"帮我选题 / 策划内容"类请求时，按序执行：

1. **锚定基础**：用「基础三问」确认账号定位、人格化、目标用户与场景。缺失则反问补全（不要凭空编）。
2. **横纵发散**：套用横纵法，列出该领域板块与细分阶段，生成 8–15 个候选选题方向。
3. **绝招加成**：对候选方向套用 10 大绝招，补充非常规角度（尤其反热点、找变化、想经历、有争议）。
4. **价值评估**：对每个候选按 5 标准打分，标出推荐度（高/中/低）与风险提示（合规/敏感/偏离定位）。
5. **产出清单**：按 `references/选题评估与工具箱.md` 的「选题清单输出模板」交付，含标题建议、角度、所用绝招、评分、风险。

## 七、输出清单格式（直接给用户）

```
【账号定位】＿＿ ｜ 【目标用户】＿＿ ｜ 【场景】＿＿

候选选题（按推荐度排序）：
1. 《标题建议》  — 角度：＿＿ ｜ 绝招：追热点/反热点/… ｜ 评分：热度x 传播x 话题x 相关x 风险x ｜ 风险：＿＿
2. …
建议优先做：第 X、Y 条（理由：＿＿）
```

## 八、反模式（不要这样做）

- ❌ 跳过基础三问直接给选题 → 选题会跑偏、不贴合账号。
- ❌ 只追热点不评估风险 → 踩合规/舆论雷。
- ❌ 一个选题想讨好所有人 → 反而没人看，盯紧核心用户痛点。
- ❌ 把"写什么"当选题 → 选题是"为什么是它 + 什么角度 + 给谁看"。
